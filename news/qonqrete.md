# QonQrete: Sandboxed Agentic IDE

**Category:** Tools & Protocols
**Key Concepts:** Agentic Coding, Local-First, File-based Memory, Containerized Sandboxes, Multi-Agent Systems

## TL;DR
**QonQrete** is a new local-first, deterministic AI coding agent framework that builds software inside hardened container environments. It operates primarily as a "construction yard" on your local machine, where a multi-agent system decomposes a high-level task into smaller sub-tasks ("briqs"), generates the code, and then actively reviews the result inside a safe sandbox. Unlike cloud-based agents that maintain opaque context windows, QonQrete is 100% file-based, meaning its entire memory, planning, and execution context live transparently on your disk.

---

## Core Architecture

QonQrete distances itself from monolithic conversational interfaces by splitting the coding process across distinct functional agents (e.g., an "InstruQtor" for planning and an "InspeQtor" for review).

1.  **Task Decomposition:** It starts by breaking down a high-level goal (`tasq.md`) into granular, manageable steps called "briqs".
2.  **Sandboxed Execution:** The generated code isn't just printed to your screen; it is actively built and executed inside a hardened Docker/Podman container (`qonqrete-qage`). This mitigates the security risks of letting an autonomous agent run arbitrary shell commands on your host OS.
3.  **File-Based Memory:** Instead of relying on a massive, ephemeral API context window that can easily suffer from "hallucination cliffs", QonQrete prepares context deterministically on disk. If the context isn't physically present in the file system, the model doesn't see it. This guarantees highly reproducible execution runs.

### Local-First, Bring Your Own Keys
QonQrete does not host a backend service. It is designed to run entirely on your own hardware, integrating directly into your JetBrains IDEs (IntelliJ, WebStorm, PyCharm) or VS Code. You plug in your own API keys for providers like OpenAI, Anthropic, Gemini, or DeepSeek, allowing you to use different models for different agent roles (e.g., using a cheaper model for planning, and a heavy reasoning model for the final code review).

## Real-World Application & Who Should Care

### 🚀 For The Performance Monsters (SOTA Seekers)
**Why you care:** QonQrete's file-based memory architecture provides deep observability into the agentic reasoning process. Because you have raw access to the intermediate planning files and "briqs" generated by the system, you can debug, fine-tune, or entirely swap out the underlying LLMs at different stages of the execution pipeline to optimize for maximum capability on complex tasks.
**Action:** Deploy the framework locally and try using a massive reasoning model (like Claude 3.7 Sonnet) strictly for the review phase ("InspeQtor"), while using a faster model for the initial code generation phase.

### 💰 For The Cost & Latency Optimizers (API Developers)
**Why you care:** Pumping massive codebases into giant context windows over and over again for every iteration of a chat conversation is incredibly expensive. QonQrete's deterministic context preparation ensures that only the strictly necessary files are sent to the model for each specific sub-task, significantly cutting down on wasted API token costs compared to naive chat-based code assistants.
**Action:** Configure the system to utilize cheaper, highly-efficient models (like Gemini 2.5 Flash) for the simpler "briq" generation tasks, reserving your expensive frontier model API calls only for the complex orchestration and final code review stages.

### 💻 For The Everyday Prompt Engineers
This topic does not apply to you. QonQrete is a local IDE-integrated multi-agent developer framework and does not affect everyday prompting strategies in conversational web interfaces like ChatGPT or Claude.
